这是什么？

TrendRadar 是一个开源的多平台热榜监控工具。它每天自动从 33+ 个平台（微博、知乎、B站、抖音、华尔街见闻、36氪、虎扑、贴吧、百度热搜…）抓取热点新闻，按您设定的关键词过滤，把匹配结果生成可视化 HTML 报告。

📌 核心价值：不再被信息流淹没。TrendRadar 替您"看"完所有平台，只推送您真正关心的话题。

☁️ 云端 24h

GitHub Actions 免费跑，电脑关机也能推送。每天 4 次自动抓取，手机/任何电脑随时查看报告。

🎯 智能匹配

支持关键词、词组、排除、正则、AI 智能分类5 种方式。词组优先级、必含词、互斥词都能配。

📈 暴涨识别

原项目内置 📈 飙升标记、🆕 新上榜标记、rank-num high 排名前 5 高亮。

修改云端频率

编辑 .github/workflows/custom-crawler.yml，改 cron 字段：

on:
  schedule:
    - cron: "0 0,4,10,14 * * *"   # ← 改这里

cron 语法：分 时 日 月 周

0 0,8,16 * * * 每天 0/8/16 点（北京时间 8/16/24 点）
0 8 * * * 每天 1 次（北京时间 16 点）
0 */2 * * * 每 2 小时一次（每天 12 次）

改完保存 → commit → push → GitHub 自动重启

A 直接编辑 frequency_words.txt

语法速查

# 注释（# 开头）
[词组名]              ← 标题行，把下面的词归成一组
关键词1                ← 普通匹配
+必含词                ← 必须同时出现
!排除词                ← 不要这些
/正则/ => 别名        ← 正则+显示名
@5                    ← 这组最多 5 条

# 组与组之间用空行分隔

实战示例

[国产AI]
/qwen|minimax|glm|deepseek|豆包|kimi|文心一言/ => 国产AI
!广告

[美国AI]
/\bOpenAI\b|\bChatGPT\b|\bSora\b/
/\bAnthropic\b|\bClaude\b/

文件位置

F:\TrendRadar\config\frequency_words.txt

B 可视化编辑器（推荐）

  1. 打开 sansan0.github.io/TrendRadar
  2. 点 "导入"，粘贴您现有的 frequency_words.txt 内容
  3. 拖拽/点选添加新词组（所见即所得）
  4. 点 "导出"，下载新的 frequency_words.txt
  5. 覆盖到 F:\TrendRadar\config\frequency_words.txt
  6. 跑一次：cd F:\TrendRadar && venv\Scripts\python.exe -m trendradar
✅ 优势：不需要记语法、不需要命令行、手机上也能编辑。

功能 3：incremental 增量模式（可选）

这是最纯粹的"黑马监控"模式：只推送上一轮没有、本轮新增的。

开启方法

编辑 config/config.yaml line 156：

report:
  mode: "incremental"          ← 改这个

三种模式对比

模式推送什么适合
current当前所有在榜新闻看全局
daily当日汇总日报
incremental只推新增黑马词监控
⚠️ 注意：incremental 模式要求系统长期连续运行才能比对"上一轮"。本地每天 1 次 + 云端每天 4 次同时用，效果最佳。

常见问题

Q：云端跑会泄露我的关键词/数据吗？

不会。仓库是 Public 公开仓库，关键词 txt 任何人都能看到。您的本地 config.yaml 里没有敏感信息（推送 webhook 等在云端读不到，但本地的也不会被 push，因为被 .gitignore 排除了）。

Q：本地和云端会不会冲突？

不会。两边独立运行，输出不同文件名（本地 output/html/latest/current.html，云端 commit 到 GitHub 的 output/）。可能本地跑了 5 分钟前云端也跑了，这是好事——您会得到 2 份最新数据。

Q：能在国内服务器跑吗？

能。GitHub Actions 在中国访问可能慢，但项目里所有数据源（newsnow）都是公开 API。如果担心稳定性，可以买一台国内 VPS，cron + Python 跑同样的脚本。

Q：报告里没有我想看的新闻怎么办？

两种可能：

  1. 关键词没匹配到 → 检查 frequency_words.txt，加更宽松的词
  2. 关键词匹配了但被过滤了 → 调 filter.method 或 max_news_per_keyword
Q：想加微信/钉钉/飞书推送怎么配？

在 config/config.yaml 找 notification: 段，填对应的 webhook：

notification:
  feishu_webhook_url: "https://open.feishu.cn/open-apis/bot/v2/hook/xxx"
  dingtalk_webhook_url: "https://oapi.dingtalk.com/robot/send?access_token=xxx"

云端配推送需要在 GitHub 仓库 Settings → Secrets and variables → Actions 加同名 secret。

Q：能同时跑多个不同关键词的实例吗？

可以：

  • 方法 1：本地用 switch-keywords.bat 切换预设（一次跑一个）
  • 方法 2：建多个 GitHub 仓库，每个跑不同关键词（但免费额度共享）
  • 方法 3：买 VPS 跑 Docker（支持并发）

对于个人使用，方法 1 足够。

Q：GitHub 仓库是 Public 安全吗？

代码本身是开源的，公开没问题。唯一注意：别把 webhook/API key 等敏感信息直接写进代码。改成本地 config/config.yaml（被 .gitignore 排除）或 GitHub Secrets（云端读）。

Q：黑马词怎么筛？

3 种方法：

  1. 看 HTML 报告 "新增热点" 区域（已开启）
  2. 报告里搜 📈 标记（飙升词）
  3. 开 AI 模式，让 AI 总结 TOP 5 黑马
Q：怎么知道云端跑得对不对？

三个检查点：

  1. Actions 页面 — 是否显示 ✅ 成功
  2. 点进 Run → 看完整 log（搜索 "new" 看新增热点数量）
  3. 看 Commits 历史 — 云端每次跑完会 commit 一次
Q：AI 模式怎么用最划算？

推荐 DeepSeek：

  1. 去 platform.deepseek.com 注册
  2. API Keys 页面创建一个 key
  3. 充 1 元钱够用几个月
  4. 填到 config/config.yaml 的 ai.api_key
  5. 改 filter.method: "ai"
